# 04 — 周报工作流

> 每周结束时，按模板输出结构化周报。

## 流程

```
回顾本周 Git log → 汇总事实 → 按 7 板块填充
  → 确保具体（不写"学习了 X"，写"用 X 做了 Y，遇到 Z 错误，靠 W 解决"）
  → 300-800 字，不凑字数
```

## 周报 7 板块

| # | 板块 | 写什么 |
|:--:|------|------|
| 1 | 本周做了什么 | 具体事情清单（装了什么工具、写了哪些代码、修了哪些 bug） |
| 2 | 学到了什么 | 新概念/工具/工作方式 + 以为对但发现错的 + aha 瞬间 + 还没搞懂的 |
| 3 | AI 协作记录 | 最爽的一次 + 最坑的一次 + 个人套路 |
| 4 | 决策记录 | 选了 X，放弃 Y，因为 Z（表格） |
| 5 | 下周打算 | 3-5 件具体事 |
| 6 | 资源沉淀 | 链接/视频/文章（标题 + URL + 一句话评价） |
| 7 | 求助/卡点 | 最想问的一个问题 + 卡了 ≥30 分钟的事 |

## 写作要求

- **真实** — 不写客套话和模板腔
- **具体** — "学到 git reset --soft HEAD~1 可以撤销 commit 但保留改动"，不是"学习了 Git"
- **简短** — 300-800 字
- **诚实** — 卡点、失败、不懂的全部写
- **可读** — 陌生人看完能复述出这周的故事

## 反例 vs 正例

| ❌ 反例 | ✅ 正例 |
|---------|--------|
| 本周学习了 Claude Code 和 git 的基础命令 | 本周装了 Claude Code，第一次跑 claude 卡在登录，原因是公司 VPN 拦了 anthropic.com。换网络后跑通 |
| 对 AI 协作有了更深的理解 | 写 CLAUDE.md 时一开始把项目目标写得太抽象（"做一个好用的工具"），后来改成具体场景，Claude 写出来的代码立刻贴合多了 |
