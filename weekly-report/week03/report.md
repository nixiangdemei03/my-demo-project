## 2. 本周学到了什么（知识 + 反思）

- **Brainstorming 的工作方式**：不是一次写对，而是 8 轮逐步校准。R1 的用户故事骨架到 R8 的供应链模型澄清，每一轮都在一个维度上消除模糊。PRD 不是"写出来的"，是"问出来的"。
- **支付模型决定了平台是工具还是生意**：R5 从"平台不碰钱"翻盘到"平台统一收银"是整个 PRD 迭代中最重要的转折。不做统一收银就没有 2% 抽成，没有抽成就没有商业模式。
- **汽配行业的供应链逻辑**：供应商提前一年备货，海外发货链条是"厂商→国际物流→海外"不经过供应商仓库。这意味着月度结算不挤压供应商现金流——之前基于 B2C 电商经验提出的"结算延迟"风险在这个行业不存在。
- **VIN 解码的务实方案**：平台不做 VIN 自动解码（一没有免费好用的全球 VIN API，二中国供应商有自己的解码工具），而是做 vin_pattern 静态匹配 + 未命中透传。这个决策节省了大量开发时间。
- 还没完全搞懂的：支付网关对接持牌机构的商务流程——这是一个非技术问题，需要 Phase 1 期间并行启动沟通。

## 3. AI 协作记录

- 最爽的一次：把"车型浏览需要车辆数据库"这个问题抛给 Claude 评估，它指出 TecDoc 年费 €5000+ 对于 v1 不可行。我问供应商怎么解决，Claude 设计了"采购商输入已知信息 → 供应商对比 VIN 和图片纠正 → 系统记录闭环"的方案。数据越用越准，零成本起步。
- 最坑的一次：R2 时 Claude 在 PRD 里写了"平台自动解析 VIN 解码车型"，技术文档写得漂亮但实际不存在免费好用的全球 VIN API。R4 时我指出供应商有自己的 VIN 解码工具，Claude 改为 vin_pattern 匹配 + 透传兜底。教训：AI 会默认假设互联网上存在它想象中 API。
- 个人套路升级：脑暴→评估→修改→再评估→确认→PRD 更新。现在不让 AI 一次写出"最终版"，而是每轮聚焦一个问题（支付→退款→VIN→供应链），逐个击破。

## 4. 决策记录

| 决策 | 方案 | 放弃 | 理由 |
|------|------|------|------|
| 支付模型 | 平台统一收银 | OCR 截图验真 | 统一收银才能抽成，OCR 验真 30% 失败率不可接受 |
| VIN 解码 | vin_pattern + 供应商兜底 | 平台自动解码 | 无免费全球 VIN API |
| 车型数据库 | 人工输入 + 纠正闭环 | 购买 TecDoc | 年费 €5000，v1 用不起 |
| 税号验证 | 天眼查 API（¥0.1/次） | 国家企业信用 API | 后者无公开 API |
| 退款决策 | 供应商自主判断检测 | 强制检测 | 供应商自己最清楚需不需要检测 |
| 客服角色 | 暂搁 | 独立建模 | 不影响 Phase 1 开发 |
