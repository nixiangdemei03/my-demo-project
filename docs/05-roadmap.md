# APEX — 实施路线图

## Phase 1: 基础平台 (Week 1-2)

- [P0] 项目脚手架：FastAPI + React + Vite，跑通 hello-world
- [P0] PostgreSQL + SQLAlchemy + Alembic 配置
- [P0] 用户注册/登录 API（JWT + bcrypt + 邮箱验证）
- [P0] 产品 CRUD API（含图片上传至 Cloudflare R2 + 车型适配绑定）
- [P0] 产品分类 API（汽配行业标准类别树）
- [P0] **搜索系统**：VIN 解析 + OEM/文字搜索 + 车型浏览三合一
- [P0] 供应商资质 OCR（营业执照税号自动验证）
- [P0] 前端：首页（搜索入口）、产品目录、产品详情、供应商详情页
- [P0] 前端：登录/注册页面
- [P0] 前端：供应商产品管理页（添加/编辑/列表）

## Phase 2: 订单与货运 (Week 3-4)

- [P0] 订单 API（创建/查询列表/状态流转）
- [P0] 货运公司 CRUD API
- [P0] 发货与追踪 API（关联货运公司+运单号+追踪事件）
- [P0] 前端：采购商下单流程
- [P0] 前端：订单详情 + 货运追踪展示
- [P1] 询盘 API + 前端消息列表
- [P1] 管理后台：用户审核、订单全局视图

## Phase 3: 完善与部署 (Week 5-6)

- [P1] 管理后台：统计看板、类别管理
- [P1] 邮件通知（注册验证、订单状态更新）
- [P1] 前端英文版（国际化）
- [P0] 前端部署至 Cloudflare Pages
- [P0] 后端部署至 Cloudflare Workers / VPS
- [P1] 集成 1-2 家货运公司 API（自动拉取追踪事件）

## 成功指标（季度制）

| 指标 | Q1 目标（上线后首个完整季度） |
|------|-----------|
| 注册供应商 | ≥ 20 |
| 上架产品 | ≥ 200 |
| 注册采购商（来自 ≥3 国家） | ≥ 50 |
| 完成订单 | ≥ 10 |
| 页面 LCP | < 3s |
| P0 端点测试通过率 | 100% |

> 指标按自然季度评估，Q1 为上线后首个完整季度。未达标则延长 Phase 3。

## 冷启动策略

开发团队需自行寻找种子用户（供应商 + 采购商），非产品功能范畴：
- 供应商端：联系国内汽配厂商/贸易商，协助完成首批上架
- 采购商端：通过行业社群、展会名录、LinkedIn 触达海外汽配批发商
- 前 40 个供应商享受 3% 低佣金（详见 PRD 结算模型）
