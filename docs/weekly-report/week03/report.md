## 2. 本周学到了什么（知识 + 反思）

- **Brainstorming 不是一次写对**：R1 用户故事骨架 → R8 供应链模型澄清，每轮只聚焦一个维度消除模糊。PRD 不是"写出来的"，是"问出来的"。
- **我之前以为支付模型只是技术选型**，这周发现它决定了平台是工具还是生意。R5 从"平台不碰钱"翻盘到"统一收银"是整个迭代最重要的转折——不碰钱就没有 2% 抽成，没有抽成就没有商业模式。
- **我最 aha 的瞬间**：担心"月度结算会挤压供应商现金流"，Claude 指出汽配行业供应商提前一年备货，海外发货链条是厂商→国际物流→海外，不经过供应商仓库。我之前基于 B2C 电商经验的假设在这个行业完全不存在。
- **我还没完全搞懂的**：支付网关对接持牌机构的商务流程——不是技术问题，需要 Phase 1 期间并行启动沟通，但找谁聊、聊什么目前空白。

## 3. AI 协作记录

- **最爽的一次**：把"车型浏览需要车辆数据库"抛给 Claude，它指出 TecDoc 年费 €5000+ 对 v1 不可行，然后设计了"采购商输入已知信息 → 供应商对比 VIN 和图片纠正 → 系统记录闭环"的方案——数据越用越准，零成本起步。
- **最坑的一次**：R2 时 Claude 在 PRD 里写了"平台自动解析 VIN 解码车型"，技术文档写得漂亮但实际不存在免费好用的全球 VIN API。R4 我发现供应商自己就有 VIN 解码工具，Claude 才改为 vin_pattern 前缀匹配 + 透传兜底。教训：**AI 会默认互联网上存在它想象出来的 API**，涉及外部数据源的方案必须亲自验证。
- **我现在的个人套路**：不让 AI 一次写出"最终版"。每轮聚焦一个问题（支付→退款→VIN→供应链），脑暴→评估→修改→确认→写入 PRD，逐个击破。8 轮下来没有一次是废的。

## 4. 决策记录

| 决策 | 选了 | 放弃了 | 理由 |
|------|------|--------|------|
| 支付模型 | 平台统一收银 | OCR 截图验真 | 统一收银才能抽成；OCR 验真 30% 失败率 |
| VIN 解码 | vin_pattern + 供应商兜底 | 平台自动解码 | 无免费全球 VIN API |
| 车型数据库 | 人工输入 + 纠正闭环 | 购买 TecDoc | 年费 €5000，v1 用不起 |
| 税号验证 | 天眼查 API（¥0.1/次） | 国家企业信用 API | 后者无公开 API |
| 退款决策 | 供应商自主判断检测 | 强制检测 | 供应商自己最清楚需不需要检测 |
