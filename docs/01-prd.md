## 数据流

### 配件搜索流程（三种入口）

```
方式 1: 车架号搜索
采购商输入 VIN → 平台用 vin_pattern 前缀匹配 product_vehicle_fits 表
            ├── 命中 → 返回适配配件列表（按类别分组）
            └── 未命中 → VIN 随询问订单透传给供应商
                         → 供应商用自有 VIN 解码工具查询车型
                         → 回复采购商具体适配什么配件

方式 2: 文字搜索
输入 OEM编号 / 配件名 → 全文搜索 products 表
                    → 匹配 name_zh / name_en / oem_number
                    → 返回结果列表

方式 3: 车型浏览（采购商输入 + 供应商纠正闭环）
采购商输入已知信息（品牌/车型/年份）→ 提交车架号给供应商
→ 供应商对比货品图片与车架号 → 纠正/补全车型信息
→ 系统录入纠正后的车型适配数据 → 下次同车型自动匹配
```

### 供应商上架流程
```
注册 → 上传资质(营业执照+税号) → OCR 提取统一社会信用代码
→ 调用天眼查/企查查 API 匹配企业名（¥0.1/次）
→ 匹配成功 → 自动 verified → 创建产品(绑定车型适配) → 上架
→ 匹配失败 → 标记人工审核
```

### 采购商下单流程（询问制）

```
采购商浏览产品 → 查看详情（含供应商信息、保修/退换政策）
→ 提交询问订单（提供 OEM 号、货品图片对比、车型信息等）
→ 供应商接入询问，核对细节，确认能否供货
→ 双方确认后生成正式订单
→ 采购商进入支付界面
→ 采购商通过平台统一收银台支付（参考支付宝海外付 / PayPal / Stripe）
→ 支付网关对接持牌机构做资金合规分账
→ 款项进入平台统一账户 → 订单状态自动变为 paid
→ 供应商确认收款到账 → 发货(关联货运+运单号)
→ 货运追踪更新 → 收货确认
```

### 支付状态模型（简化）

```
未支付 (unpaid)
  └── 采购商下单但未完成支付 → 订单保持 unpaid
        → 48h 后自动提醒采购商 → 72h 未付自动取消

已支付 (paid)
  └── 款项进入平台账户 → 自动确认
        → 供应商发货 → 收货后进入结算周期
```

### 平台结算模型

```
每月 1 日结算上月已收货订单
  └── 月总销售额 = Σ 所有已收货订单金额
  └── 供应商利润 = 月总销售额 × 10%（行业默认配件利润率）
  └── 平台抽成 = 供应商利润 × 20% = 月总销售额 × 2%
  └── 供应商实际到账 = 月总销售额 - 平台抽成 = 月总销售额 × 98%
  └── 平台出具结算单 → 供应商确认 → 批量打款
```

**抽成示例**：
```
某供应商月销售额 $50,000，月中发生 1 笔 $3,000 退款
退款流程：采购商退货 → 供应商决定是否需要检测报告
         ├── 不需要 → 供应商直接对接厂家退货 → 确认收到退货 → 进入退款
         └── 需要 → 供应商收货检测 → 提供检测报告/定损 → 确认定损结果
         → 无争议 → 从供应商当月结算中扣除 $3,000
月结算基数 = $50,000 - $3,000 = $47,000
  → 供应商利润 = $47,000 × 10% = $4,700
  → 平台抽成 = $4,700 × 20% = $940（相当于净销售额的 2%）
  → 供应商实际收到 = $47,000 - $940 = $46,060
```

### 供应商拒绝流程

```
供应商收到询问订单 → 接入沟通细节 → 确认能否供货
→ 若无法供货（缺货/型号不符/价格变动）→ 拒绝并附原因
→ 采购商收到通知 → 平台推荐同类产品
```

### 纠纷 / 退款流程

```
采购商收货后不满意 → 查看产品页供应商写明的保修/退换政策
→ 采购商与供应商协商退货
   ├── 供应商不需要检测报告 → 直接对接厂家退货 → 确认收到退货 → 进入退款
   └── 供应商需要检测报告 → 收货检测 → 出具报告/定损 → 确认后进入退款
→ 退款金额从供应商当月结算中扣除
→ 若协商失败 → 任一方向平台申请客服介入
        → 平台客服查看订单记录、双方凭证
        → 做出调解建议 → 双方执行
```

### 图片上传流程
```
前端上传 → POST /api/products/:id/images → 后端生成唯一文件名
→ 上传 Cloudflare R2 → 保存 URL 到 product_images → 返回 URL
```
