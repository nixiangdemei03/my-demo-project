# 5905_Lab — Lab 知识点分析与刷题生成

## 触发方式

```
/5905_Lab <文件路径>
```

或直接说 "分析这个 lab" 并提供文件路径。

## 功能

1. SSH 进入学生账户获取指定 lab 文件
2. 分析文件中的**知识点**、**易错点**、**易考察点**
3. 生成知识点总结 `.md` → 保存到 `~/Desktop/5905/lab/`
4. 将原题文件复制到 `~/Desktop/5905/刷题/`
5. 根据本次知识点，生成 **10 道练习题**（简单 3 道、中等 4 道、难题 3 道）→ 保存到 `~/Desktop/5905/刷题/`

## 执行流程

### Step 1: 确认 SSH 信息

向用户确认以下信息（如未提供则逐一询问）：

```
SSH 用户名：  （如：student01）
SSH 主机地址：（如：lab.example.com）
SSH 端口：    （默认 22）
文件路径：    （如：/home/student01/labs/week03/lab3.c）
身份验证方式：密码 / SSH Key
```

### Step 2: 获取文件

```bash
# 使用密码
sshpass -p "<password>" scp -P <port> <user>@<host>:<file_path> ./temp_lab_file

# 使用 SSH Key
scp -P <port> <user>@<host>:<file_path> ./temp_lab_file
```

**权限失败时**：提示用户检查 SSH 凭据，不要反复尝试。

### Step 3: 分析文件内容

读取 `temp_lab_file`，从以下维度分析：

#### 知识点（Knowledge Points）
- 代码/文本中涉及的**核心概念**和**技术点**
- 每个知识点标注：名称 + 所在行号 + 简要说明
- 输出格式：
  ```markdown
  ## 知识点清单
  | # | 知识点 | 行号 | 说明 |
  |---|--------|:----:|------|
  | 1 | 指针运算 | L12-18 | 通过指针遍历数组 |
  ```

#### 易错点（Common Mistakes）
- 代码中**容易写错**或**编译通过但逻辑错误**的地方
- 每种错误标注：位置、错误类型、正确写法
- 输出格式：
  ```markdown
  ## 易错点
  ### 错误 1: 数组越界
  - 位置: L25
  - 错误写法: `for(i=0; i<=n; i++)`
  - 正确写法: `for(i=0; i<n; i++)`
  - 解释: 数组长度为 n 时，最大索引为 n-1
  ```

#### 易考察点（Exam Points）
- 基于此代码**可能的变化题型**
- 可能被问到的**概念题**或**改错题**
- 输出格式：
  ```markdown
  ## 易考察点
  | # | 考察形式 | 考察内容 | 难度 |
  |---|----------|----------|:--:|
  | 1 | 改错题 | L20 的边界条件 | 中 |
  | 2 | 概念题 | 指针与数组的关系 | 易 |
  ```

### Step 4: 生成知识点总结

输出到 `~/Desktop/5905/lab/<原文件名>_analysis.md`

```markdown
# Lab Analysis — <文件名>

> 分析日期：<日期>
> 来源文件：<文件路径>

## 知识点清单
...

## 易错点
...

## 易考察点
...

## 快速自测
- [ ] 我能解释每个知识点的原理吗？
- [ ] 我能指出每处易错点的问题吗？
- [ ] 我能独立写出易考察点的答案吗？
```

### Step 5: 复制原题

```bash
# 创建目录（如不存在）
mkdir -p ~/Desktop/5905/刷题

# 复制原题文件
cp temp_lab_file ~/Desktop/5905/刷题/<原文件名>
```

### Step 6: 生成练习题

根据分析出的知识点，生成 **10 道题目**：

| 难度 | 数量 | 题型 | 说明 |
|------|:--:|------|------|
| 简单 | 3 | 概念填空 / 选择 | 检验基础知识记忆 |
| 中等 | 4 | 代码改错 / 代码补全 | 检验理解和应用 |
| 难题 | 3 | 完整编写 / 综合题 | 检验综合运用能力 |

输出到 `~/Desktop/5905/刷题/<原文件名>_practice.md`

```markdown
# 练习题 — <文件名>

> 共 10 题 | 简单 3 | 中等 4 | 难题 3

---

## 一、简单题（3 道）

### Q1. [知识点名称]
题目描述...

<details>
<summary>答案</summary>
...
</details>

---

## 二、中等题（4 道）

### Q4. [知识点名称]
题目描述...

---

## 三、难题（3 道）

### Q8. [综合题]
题目描述...

---

## 参考答案

### Q1 答案
...

### Q2 答案
...

...
```

**题目质量要求**：
- 每题必须对应一个已分析出的知识点
- 简单题检验记忆和基础理解
- 中等题需要代码改动或分析
- 难题需要综合 2+ 个知识点
- 每题必须有独立答案（折叠显示）

### Step 7: 清理

```bash
rm -f temp_lab_file
```

并向用户报告：

```
✅ 分析完成
📁 知识点总结: ~/Desktop/5905/lab/<文件名>_analysis.md
📁 原题文件:   ~/Desktop/5905/刷题/<原文件名>
📁 练习题:     ~/Desktop/5905/刷题/<原文件名>_practice.md
📊 知识点: <N> 个 | 易错点: <M> 个 | 易考察点: <K> 个 | 练习题: 10 道
```

## 安全规则

1. SSH 密码**不记录**到任何文件，仅用于本次 `sshpass` 命令
2. 每次连接成功后立即关闭会话
3. 不修改学生账户上的任何文件
4. 不保留远程文件的本地缓存（`temp_lab_file` 完成后删除）
5. 生成的题目保存到 `~/Desktop/` 而非项目仓库

## 依赖

本 skill 需要以下工具已安装：

| 工具 | 用途 | 安装 |
|------|------|------|
| `sshpass` | 密码方式 SSH 传输 | `apt install sshpass` / `brew install sshpass` |
| `scp` | SSH 文件传输 | 系统自带 |
| `ssh-keygen` | SSH Key 方式 | 系统自带 |

如 `sshpass` 未安装，skill 自动降级为交互式提问（让用户在终端手动 scp）。
