# /5905

## 这个 skill 做什么

SSH 进入学生账户获取指定 lab 文件，分析文件中的知识点、易错点、易考察点，生成知识点总结和 10 道练习题（简单 3 + 中等 4 + 难题 3），输出到 `~/Desktop/5905/`。

## 输入

- 用户的 SSH 凭据（用户名、主机地址、端口、密码或 Key）
- 目标文件的远程路径（如 `/home/student01/labs/week03/lab3.c`）

## 输出格式

### 知识点总结 → `~/Desktop/5905/lab/<原文件名>_analysis.md`

```markdown
# Lab Analysis — <文件名>
> 分析日期：<日期> | 来源文件：<文件路径>

## 知识点清单
| # | 知识点 | 行号 | 说明 |
|---|--------|:----:|------|
| 1 | 指针运算 | L12-18 | 通过指针遍历数组 |

## 易错点
### 错误 1: 数组越界
- 位置: L25
- 错误写法: `for(i=0; i<=n; i++)`
- 正确写法: `for(i=0; i<n; i++)`
- 解释: 数组长度为 n 时，最大索引为 n-1

## 易考察点
| # | 考察形式 | 考察内容 | 难度 |
|---|----------|----------|:--:|
| 1 | 改错题 | L20 的边界条件 | 中 |

## 快速自测
- [ ] 我能解释每个知识点的原理吗？
- [ ] 我能指出每处易错点的问题吗？
- [ ] 我能独立写出易考察点的答案吗？
```

### 练习题 → `~/Desktop/5905/刷题/<原文件名>_practice.md`

共 10 题（简3 / 中4 / 难3），每题对应已分析出的知识点，答案折叠显示。

### 原题副本 → `~/Desktop/5905/刷题/<原文件名>`

## 执行流程

1. **确认 SSH 信息** — 如未提供，逐一询问用户名/主机/端口/文件路径/认证方式
2. **获取文件** — 用 `SSHPASS` 环境变量 + `sshpass -e` 传输（密码不出现在命令行），SSH Key 则用 `scp`
3. **分析文件** — 从知识点（核心概念+行号+说明）、易错点（位置+错误写法+正确写法+解释）、易考察点（考察形式+内容+难度）三个维度分析
4. **生成知识点总结** — 输出 analysis.md
5. **复制原题** — `cp` 到刷题目录
6. **生成练习题** — 10 道题，每题独立答案（`<details>` 折叠）
7. **清理** — `rm -f temp_lab_file`，报告产出

## 必须遵守

- SSH 密码通过 `SSHPASS` 环境变量传递（`sshpass -e`），不出现于命令行参数；使用后立即 `unset`
- 每次连接成功后立即关闭会话，不修改学生账户上的任何文件
- 不保留远程文件的本地缓存（`temp_lab_file` 完成后删除）
- 生成的题目保存到 `~/Desktop/` 而非项目仓库
- 权限失败时提示用户检查 SSH 凭据，不反复尝试
- 每题必须对应一个已分析出的知识点；简单题检验记忆、中等题需要代码分析、难题综合 2+ 知识点

## 依赖

| 工具 | 用途 | 安装 |
|------|------|------|
| `sshpass` | 密码方式 SSH 传输 | `apt install sshpass` / `brew install sshpass` |
| `scp` | SSH 文件传输 | 系统自带 |

> 如 `sshpass` 未安装，自动降级为交互式提问（让用户在终端手动 scp）。

## 反模式

- 把密码写在命令行参数里（`sshpass -p "xxx"`）→ 用 `SSHPASS` 环境变量
- 跑题生成与 lab 文件无关的通用题目
- 把 lab 文件或分析结果提交到项目仓库
- 分析空洞（"这段代码用了循环"）→ 必须标注行号+具体说明
