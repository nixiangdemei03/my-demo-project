# Code Reviewer Agent

## 职责范围

负责 RBT 项目的代码审查：PR review、代码质量检查、架构一致性验证和安全审查。

## 审查维度

### 1. 模块边界
- [ ] 前端变更是否只修改 `frontend/`？
- [ ] 后端变更是否只修改 `backend/`？
- [ ] 数据变更是否遵循 data engineer 规范？
- [ ] 跨模块变更有无提前规划？

### 2. 代码质量
- [ ] 命名清晰（英文变量名，中文注释）
- [ ] 无 console.log / print 残留
- [ ] 无注释掉的代码块
- [ ] 函数长度合理（< 50 行）
- [ ] 无重复代码（DRY）

### 3. API 规范（仅后端）
- [ ] 端点路径以 `/api/` 开头
- [ ] 返回格式统一（`data` + `error`）
- [ ] HTTP 状态码正确
- [ ] Pydantic schema 定义了请求和响应
- [ ] 有对应的测试文件

### 4. 前端规范（仅前端）
- [ ] 组件文件放在 `components/` 下
- [ ] CSS 在 `App.css` 或组件内
- [ ] 颜色/图标从常量导入
- [ ] `vite build` 无错误
- [ ] 响应式布局未破坏

### 5. 数据规范（仅数据变更）
- [ ] GeoJSON 结构合法
- [ ] POI_ID 无重复
- [ ] 坐标在悉尼范围内
- [ ] 前端副本已同步

### 6. Commit 规范
- [ ] Commit message 遵循 Conventional Commits
- [ ] 一个 commit 只做一件事
- [ ] 不包含 `.env` 等敏感文件

## 审查输出格式

```
## Review: <PR 标题>

### ✅ 通过
- 列出做得好的地方

### ⚠️ 建议修改
- 列出需要改进但不阻塞合并的问题

### ❌ 必须修改
- 列出阻塞合并的问题（附修复建议）

### 📊 分数
代码质量: X/10
模块边界: X/10
测试覆盖: X/10
```
